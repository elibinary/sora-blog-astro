## 三、内容迁移策略

### 3.1 Markdown 格式转换

#### Hexo → Astro 注意事项

```markdown
# Hexo Front Matter
---
title: 文章标题
date: 2017-06-01 10:00:00
categories:
  - 技术
  - 数据库
tags:
  - MySQL
  - 索引
---

# Astro (AstroPaper) Front Matter
---
title: 文章标题
pubDate: 2017-06-01 10:00:00
category: 数据库
tags:
  - MySQL
  - 索引
draft: false
---
```

#### 转换要点
- `date` → `pubDate`
- `categories` (数组) → `category` (单值，AstroPaper 默认)
- 检查代码块语法高亮兼容性
- 处理 Hexo 特有标签（如 `{% codeblock %}` → ``` 代码块）

### 3.2 图片资源迁移

#### 现状分析
- 旧博客图片可能存储在：
  - 本地 `source/images/` 目录
  - 外部图床（需检查链接有效性）

#### 迁移方案
1. **收集所有图片**：扫描所有 Markdown 文件中的图片引用
2. **统一存储**：放入 Astro 项目的 `public/images/` 或 `src/assets/`
3. **更新引用**：批量替换图片路径
4. **优化处理**：使用 Astro 的 Image 组件进行优化

```bash
# 查找所有图片引用
grep -r "!\[" source/_posts/ | grep -oE "images/[^)]+"
```

### 3.3 URL 重定向（SEO 保护）

#### 旧 URL 格式
```
http://elibinary.com/:year/:month/:day/:title/
例：http://elibinary.com/2017/06/01/mysql-index-optimization/
```

#### 新 URL 格式（AstroPaper 默认）
```
/posts/:slug/
例：/posts/mysql-index-optimization/
```

#### 重定向方案

1. **创建重定向文件** (`public/_redirects` 用于 Netlify，或 `vercel.json` 用于 Vercel)：

```
# Netlify _redirects
/2017/06/01/mysql-index-optimization/  /posts/mysql-index-optimization/  301
```

2. **或使用 Astro 中间件**：

```javascript
// middleware.ts
export async function onRequest(context, next) {
  const url = new URL(context.request.url);
  const oldPattern = /^\/(\d{4})\/(\d{2})\/(\d{2})\/(.+)\/$/;
  const match = url.pathname.match(oldPattern);
  if (match) {
    const newSlug = match[4];
    return Response.redirect(`/posts/${newSlug}`, 301);
  }
  return next();
}
```

3. **生成重定向映射表**：
   - 遍历所有旧文章，生成 slug 映射
   - 批量创建重定向规则

### 3.4 保留历史文章的价值

1. **添加迁移说明**：在旧文章顶部添加说明
   ```markdown
   > 📝 本文迁移自旧博客，原文发布于 2017 年。部分内容可能需要更新。
   ```

2. **设置"归档"标签**：标记为历史内容

3. **选择性更新**：
   - 技术原理类：基本不需要更新
   - 工具/版本相关：添加更新说明
   - 过时内容：考虑归档或删除

## 四、内容优化建议

### 4.1 内容分类体系

#### 建议分类结构

```
📚 技术文章
├── 💾 数据库
│   ├── MySQL
│   └── Redis
├── 🔧 后端开发
│   ├── Ruby
│   └── Rails
├── 🌐 Web 开发
│   ├── 前端基础
│   ├── HTTP/网络
│   └── 安全认证
├── 🧮 算法与数学
│   ├── 数据结构
│   └── 线性代数
└── 🛠️ 工程实践
    └── 系统设计
```

#### Astro 配置建议

```typescript
// src/content/config.ts
export const blogSchema = z.object({
  title: z.string(),
  pubDate: z.date(),
  category: z.enum(['数据库', '后端开发', 'Web 开发', '算法与数学', '工程实践']),
  tags: z.array(z.string()),
  // ...
});
```

### 4.2 标签系统优化

#### 标签规范
- **统一大小写**：建议首字母大写或全小写
- **避免重复**：MySQL vs mysql → 统一为 MySQL
- **控制数量**：每篇文章 3-5 个标签
- **建立标签云**：在侧边栏或独立页面展示

#### 建议标签体系
```
# 技术栈
MySQL, Redis, Ruby, Rails, JavaScript

# 主题
索引，查询优化，源码分析，数据结构，认证授权

# 系列
MySQL 系列，Redis 系列，Rails 源码，线性代数
```

### 4.3 系列文章组织

#### 现有系列识别
- MySQL 查询优化系列 (I, II, III)
- Redis 内部机制系列 (I-IX)
- Rails 源码分析系列
- 线性代数系列

#### 组织方案

1. **系列索引页**：为每个系列创建导航页
   ```markdown
   # MySQL 查询优化系列
   
   本系列深入探讨 MySQL 查询优化的各个方面：
   
   1. [MySQL 查询优化 (一)：基础概念](/posts/mysql-query-optimization-i/)
   2. [MySQL 查询优化 (二)：执行计划分析](/posts/mysql-query-optimization-ii/)
   3. [MySQL 查询优化 (三)：实战案例](/posts/mysql-query-optimization-iii/)
   ```

2. **系列元数据**：
   ```yaml
   series: MySQL 查询优化
   seriesOrder: 1  # 系列中的第几篇
   ```

3. **系列导航组件**：在文章页面显示上一篇/下一篇

## 六、实施建议

### 6.1 迁移工作流程

```
1. 备份旧博客仓库
   ↓
2. 批量转换 Front Matter
   ↓
3. 迁移图片资源
   ↓
4. 配置 URL 重定向
   ↓
5. 逐篇检查文章内容
   ↓
6. 设置分类和标签
   ↓
7. 部署测试
   ↓
8. 验证重定向
```

### 6.2 工具推荐

- **Front Matter 转换**：编写脚本批量处理
- **图片迁移**：使用 `grep` + `sed` 批量替换路径
- **重定向生成**：根据文章列表自动生成映射
- **内容检查**：使用 Astro 的 Content Collections 验证

### 6.3 注意事项

1. **保留 Git 历史**：考虑将旧博客作为子模块或独立分支保留
2. **SEO 监控**：迁移后使用 Google Search Console 监控索引状态
3. **用户反馈**：添加反馈渠道，收集死链报告
4. **定期备份**：建立定期备份机制
